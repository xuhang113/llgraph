# llgraph 项目结构（开发文档）

本文面向**维护与二次开发**：说明仓库目录、Python 子包划分、运行时调用链，以及「改功能该找哪」。

更细的模块职责与扩展说明见 [模块说明.md](模块说明.md)；使用与排障见 [操作手册.md](操作手册.md)。

---

## 1. 仓库根目录

```
llgraph/                          # 项目根
├── pyproject.toml                # 包定义、依赖、控制台入口 llgraph = llgraph.main:main
├── README.md
├── docs/                         # 文档（本文件、模块说明、操作手册等）
├── examples/                     # agent.json、mcp.json 示例
├── scripts/                      # 维护脚本（非运行时）
│   ├── check_model_dispatch_compat.py   # 多模型出站协议兼容测试
│   ├── probe_prompt_cache.py            # Prompt Cache 探针
│   ├── migrate_session_canonical_v2.py  # 会话 jsonl 迁移
│   └── reorganize_packages.py           # 包结构整理（一次性）
└── llgraph/                      # Python 包（见下文）
```

**安装入口**：`pip install -e .` 后通过 `llgraph` 或 `python -m llgraph` 启动。

---

## 2. Python 包总览

`llgraph/` 包内按**业务域**分子目录；导入统一为 `llgraph.<子包>.<模块>`。

```
llgraph/
├── main.py                 # CLI 入口：参数解析、构建 Agent、进入交互
├── __main__.py             # python -m llgraph
├── __init__.py
│
├── core/                   # Agent 编排、Gateway LLM、工具实现
├── context/                # 上下文注入、消息规范、压缩、出站窗口
├── session/                # 会话持久化、切换、编辑记录、历史检索
├── config/                 # 环境变量、agent.json、各 feature settings
├── loaders/                # Rules / Skills / Commands / Thought 加载
├── commands/               # 斜杠命令（/review、/trace、/survey …）
├── survey/                 # 模型 <<<survey>>> 块解析与终端确认向导
├── display/                # trace 输出、执行日志、终端/TUI 模式开关
├── cli/                    # llgraph index / search 子命令实现
│
├── code_index/             # 代码向量索引（LanceDB、切块、检索）
├── terminal/               # 经典终端交互（readline、survey 向导、原地重绘）
└── ui/                     # Textual TUI（--ui tui）
```

## 3. 子包说明

### 3.1 `core/` — Agent 与工具

ReAct Agent 的构建与执行，以及所有 LangGraph 可调工具。

| 模块 | 职责 |
|------|------|
| `agent.py` | **核心**：`build_agent()`、`invoke_agent()`，组装 LLM + 工具 + 系统提示 |
| `agent_config.py` | 从 `agent.json` 读取 Agent 级配置 |
| `agent_session.py` | 单次 CLI 进程的会话上下文（thread、resume 等） |
| `llm.py` | 创建指向 Gateway 的 `ChatAnthropic` |
| `llm_settings.py` | 模型名解析（CLI `--model` / env / agent.json） |
| `gateway_models.py` | 网关模型列表与能力探测 |
| `gateway_kimi_patch.py` | Kimi 多轮 tool + thinking 协议补丁 |
| `prompt_cache.py` / `prompt_cache_settings.py` | Anthropic 兼容 Prompt Cache 断点 |
| `model_context_window.py` | 各模型上下文窗口上限 |
| `tools.py` | **工具注册入口**：`get_agent_tools()` |
| `filesystem_tools.py` | read/list/grep/write/search_replace 等文件工具 |
| `shell_tools.py` | 受控 shell 执行 |
| `web_search_tools.py` | Tavily 联网搜索 |
| `mcp_tools.py` | MCP stdio 工具桥接 |
| `code_index_tools.py` | `search_code_*` 语义检索工具 |
| `workspace.py` | 工作区路径安全边界 |
| `checkpointer_factory.py` | LangGraph MemorySaver 等 |
| `write_failure_tracker.py` | 写工具连续失败提醒 |

```python
from llgraph.core.agent import build_agent, invoke_agent
from llgraph.core.tools import get_agent_tools
```

## 4. 运行时调用链

```
llgraph main.py
    │
    ├─ config.config.load_llgraph_env()
    ├─ core.agent.build_agent()
    │     ├─ core.llm.create_gateway_llm()
    │     ├─ core.tools.get_agent_tools()
    │     ├─ loaders.* / context.context_builder
    │     └─ LangGraph create_react_agent
    │
    └─ terminal.session.run_terminal_session()  或  ui.app.run_tui_session()
          │
          ├─ 读入用户输入 → commands.meta_commands（斜杠命令）
          ├─ survey.survey_prompt（需确认时）
          └─ core.agent.invoke_agent / trace_display.stream_agent_turn
                │
                ├─ session.session_file_store（落盘 messages.jsonl）
                ├─ context.*（压缩、spill、dispatch）
                └─ Gateway API
```

## 5. 依赖分层（建议）

新增代码时尽量遵守**单向依赖**，避免子包互相绕圈：

```
config          ← 最底层，不依赖业务子包
loaders         ← 仅依赖 config、workspace 路径
session         ← config、context（少量）
context         ← config、session（文件存储）
core            ← config、context、session、loaders、survey、display
commands        ← core、context、session、config、cli、code_index…
terminal / ui   ← core、commands、display、survey、session
main            ← 以上全部（装配层）
```

**规则简述**：

- `config/` 不要 import `core/`、`commands/`。
- 工具实现放 `core/`，不要在 `tools.py` 里堆业务逻辑。
- 斜杠命令放 `commands/`，不要在 `main.py` 里加分支。
- 仅 CLI 用的放 `cli/`；Agent 工具放 `core/*_tools.py`。

## 6. 常见改动该改哪

| 目标 | 位置 |
|------|------|
| 新增 Agent 工具 | `core/tools.py` 注册 + 新建 `core/*_tools.py` 或扩展现有文件 |
| 改系统提示 / ReAct 流程 | `core/agent.py` |
| 换模型 / Gateway 行为 | `core/llm.py`、`core/gateway_*.py` |
| 新斜杠命令 | `commands/meta_commands.py` + `help_text.py` |
| 会话落盘格式 | `context/message_canonical.py`、`session/session_file_store.py` |
| 压缩 / spill / 出站窗口 | `context/context_compressor.py`、`context_spill.py`、`context_dispatch_window.py` |
| 终端交互 / survey UX | `terminal/`、`survey/` |
| TUI | `ui/` |
| agent.json 新字段 | `config/edit_settings.py` + 对应 `*_settings.py` |
| 索引算法 | `code_index/` |
| 维护脚本 | `scripts/`（不打包进运行时） |
