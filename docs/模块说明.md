# llgraph 模块说明

本文说明项目中各文件/模块的职责、调用关系与扩展方式。

**产品背景**：llgraph 在终端复刻 **Cursor Agent** 的核心链路（ReAct + 工作区工具 + Rules/Skills + 索引 + MCP + 上下文治理），实现细节见 [cursor-agent.md](cursor-agent.md)。

> **开发维护**：目录与子包划分、调用链、常见改动入口见 [项目结构.md](项目结构.md)。

## 包结构（按业务域）

`llgraph/` 已按职责拆分子包，请直接使用 `llgraph.<子包>.<模块>` 导入。

```
llgraph/
├── main.py              # CLI 入口
├── core/                # Agent 编排、Gateway LLM、工具
├── context/             # 上下文构建、消息规范、压缩、出站窗口
├── session/             # 会话持久化、切换、编辑记录、历史检索
├── config/              # 环境变量、agent.json、各 feature settings
├── loaders/             # Rules / Skills / Commands / Thought 加载
├── commands/            # 斜杠命令（/review、/trace、help）
├── survey/              # 交互式 survey、编辑确认
├── display/             # 过程追踪、执行日志、终端样式
├── cli/                 # llgraph index / search 子命令
├── code_index/          # 向量索引（已有）
├── terminal/            # 经典终端交互
```

| 子包 | 典型 import | 职责 |
|------|-------------|------|
| `core` | `from llgraph.core.agent import build_agent` | ReAct Agent、LLM、工具注册 |
| `context` | `from llgraph.context.context_session import ContextSession` | 压缩、落盘、消息修链 |
| `session` | `from llgraph.session.session_edits import SessionEditTracker` | jsonl 持久化、/changes |
| `config` | `from llgraph.config.config import load_llgraph_env` | Gateway 凭据、agent.json |
| `loaders` | `from llgraph.loaders.rules_loader import load_rules` | Rules/Skills 注入 |
| `commands` | `from llgraph.commands.meta_commands import handle_meta` | 斜杠命令 |
| `survey` | `from llgraph.survey.survey_prompt import extract_survey_block` | 多步确认向导 |
| `display` | `from llgraph.display.trace_display import TraceSession` | trace / 耗时 / 颜色 |
| `cli` | `from llgraph.cli.index_cli import run_index` | 索引 CLI |

迁移脚本：`scripts/reorganize_packages.py`（一次性整理，已执行）。

## 整体架构

```
用户 / CLI (main.py)
        │
        ▼
   core/agent.py ──► build_agent() / invoke_agent()
        │
        ├── config/config.py   加载 Gateway 环境变量
        ├── core/llm.py        创建 ChatAnthropic（指向 AI Gateway）
        └── core/tools.py      Agent 可调用的工具
                │
                ▼
        LangGraph create_react_agent
        （LLM ↔ 工具 循环，直至给出最终回复）
                │
                ▼
        AI Gateway（Anthropic 兼容 API）
```

## Python 包内模块（`llgraph/`）

> 下列文件名若未写子包前缀，实现位于对应子包；例如 `config.py` → `llgraph/config/config.py`。

### `config/config.py` — 配置加载

| 职责 | 说明 |
|------|------|
| 环境变量来源 | `~/.config/llgraph/llgraph.env`，可选项目根 `.env` 覆盖 |
| 变量名 | `LLGRAPH_API_BASE_URL`、`LLGRAPH_API_KEY`、`LLGRAPH_MODEL` |
| 对外接口 | `load_llgraph_env()`；`get_llgraph_settings()` → `base_url` / `api_key` / `model` |

**不读取** `claude-code.env` 或 `ANTHROPIC_*`。Embedding remote 可复用 `LLGRAPH_*` 或 `EMBEDDING_*`。

---

### `llm.py` — 大模型客户端

| 职责 | 说明 |
|------|------|
| Gateway 连接 | `ChatAnthropic` + `LLGRAPH_API_BASE_URL` / `LLGRAPH_API_KEY` |
| 兼容性处理 | 对 Gateway 返回的 `usage.cache_creation` 中 `null` 字段做 patch，避免 LangChain 解析报错 |
| 参数约定 | 不显式设置 `temperature=0.2`（网关会 400）；`max_tokens` 默认 16384，见 `agent.json` → `llm.max_tokens` |

**对外接口**：`create_gateway_llm(workspace)` → 返回配置好的 `ChatAnthropic` 实例。

**何时改**：切换 LLM 实现、调整 `max_tokens`、或网关契约变更时修改。

---

### `tools.py` — 工具注册

| 职责 | 说明 |
|------|------|
| 通用工具 | `get_current_utc_time` |
| 文件工具 | 通过 `filesystem_tools.create_filesystem_tools` 按工作区注入 |
| 对外接口 | `get_agent_tools(workspace_root, allow_write)` |

### `workspace.py` — 工作区安全边界

解析相对路径、禁止路径穿越、跳过 `.git` / `node_modules` 等目录，限制单文件读取大小。

### `filesystem_tools.py` — 文件类工具

| 工具 | 只读 | 说明 |
|------|------|------|
| `list_directory` | ✓ | 列目录 |
| `search_files` | ✓ | 按文件名 / glob 查找 |
| `grep_files` | ✓ | 内容正则/子串搜索 |
| `read_file` | ✓ | 读文件，支持行号范围 |
| `write_file` | 仅 `-w` | 创建或覆盖文件；`path`+`content` 必填 |
| `append_file` | 仅 `-w` | 追加到文件末尾；长文档分块写入 |
| `search_replace` | 仅 `-w` | 局部替换；改已有文件优先 |

**何时改**：新增业务能力（查库、调 HTTP 等）时在 `tools.py` / 新模块扩展。

---

### `agent.py` — Agent 编排（核心）

| 职责 | 说明 |
|------|------|
| 图构建 | 使用 LangGraph 预置 `create_react_agent`，组合 LLM + 工具 + 系统提示词 |
| 会话记忆 | 可选 `MemorySaver` + `thread_id`（`--memory` 时启用） |
| 执行封装 | `invoke_agent()`：传入用户消息，返回助手最后一条文本回复 |

| 函数 | 作用 |
|------|------|
| `build_agent(with_memory=False)` | 编译并返回可执行的 LangGraph Agent |
| `invoke_agent(agent, user_message, ...)` | 执行一轮对话，解析 `messages` 中的最终内容 |

**何时改**：更换 Agent 模式（如多节点 StateGraph、人工审批 interrupt）时主要改此文件。

---

### `trace_display.py` — 过程展示与耗时

通过 `agent.stream(stream_mode=["updates", "messages"])` 实时打印：

- 用户消息、模型规划（tool_calls）、工具执行结果预览
- 助手回复（支持流式 token）
- 每步与本轮总耗时；TTY 下带颜色

`/trace stats`：估算 `messages_tokens`、落盘字节、spill 次数（P6）。

---

### `context_spill.py` — 动态上下文落盘（P6）

| 职责 | 说明 |
|------|------|
| 阈值落盘 | 工具返回超过 `tool_result_max_chars` 时写入 `.llgraph/context/tool-results/` |
| 指针消息 | 返回路径、行数、末尾预览；模型用 `read_file` / `grep_files` 按需读取 |
| 工具包装 | `apply_spill_to_tools()` 统一挂接内置工具与 MCP |
| 统计 | `/trace stats` 展示 token 与落盘体积 |

配置：`agent.json` → `context.spill_*`。

---

### 会话上下文与历史（P3 / P6）

**详细说明**：[会话上下文与历史.md](会话上下文与历史.md)（格式、加载流程、Token 节约、压缩 Tier1～3、出站窗口、`search_session_history`）。

| 模块 | 职责 |
|------|------|
| `context_settings.py` | 解析 `agent.json` → `context`（`compress_strategy`、dispatch、spill、`_docs`） |
| `context_compressor.py` | `/compress`、自动压缩、会话归档 jsonl |
| `context_dispatch_window.py` | `apply_dispatch_window_trim`：固定 N 轮或 auto 按 token 扩展 |
| `context_continuity.py` | 续写/重写 pin → `<workspace-context>`（已改路径、近期 read、上轮摘要） |
| `incremental_context.py` | 增量 Tool 掩码、自动压缩阈值 |
| `session_file_store.py` | 对话持久化 `messages.jsonl`（canonical v2） |
| `message_canonical.py` | 跨模型落盘形态 |
| `message_dispatch_profile.py` / `chat_history_repair.py` | 落盘 vs 出站修链；`tool_call_id` canonical 匹配 |
| `message_normalize.py` | 合并 system、prompt normalizer |
| `session_history_search.py` / `session_history_tools.py` | 按需检索全历史 |
| `user_storage.py` | 用户目录路径、`migrate_legacy_workspace_session_dir` |
| `session_registry.py` | 列举会话、空壳过滤 |
| `session_meta.py` / `session_delete.py` | 标题、删除 |
| `session_manifest.py` | manifest 侧车与置顶 `<session-manifest>` |
| `conversation_anchor.py` | 结构化锚点（Tier2/3） |
| `context_spill.py` | 超大 tool 结果落盘 |
| `runtime_context.py` | 当前 `thread_id`（供历史检索工具） |
| `prompt_cache.py` / `prompt_cache_settings.py` | `cache_control` 断点（稳定 system + tools + 对话尾） |

压缩前会将待压缩段中的超长 `ToolMessage` 替换为简短指针。会话记忆仅 jsonl + 进程内 MemorySaver，无 SQLite。

### `session_edits.py` — 会话变更记录（P7）

`SessionEditTracker`：`search_replace` / `write_file` 记账、`/changes`、`/diff` 快照。

落盘：`~/.llgraph/context/<工作区>/sessions/<id>/edits.jsonl`、`snapshots/`（与 `messages.jsonl` 同目录）。

---

### `mcp_config.py` / `mcp_tools.py` — MCP（P1）

仅读取 `.llgraph/mcp.json`（**不复用** `.cursor/mcp.json`）。stdio 长连接，工具名 `mcp__<server>__<tool>`。

---

### `commands_loader.py` — 自定义命令（P2）

`.llgraph/commands/*.md`（frontmatter + prompt）；`/commands` 列表。

---

### `review_command.py` — 代码评审（P4）

`/review [主题]`：评审 `/changes` 文件 + git diff，落盘 `~/llgraph-review/`。

---

### `code_index/index_watch.py` — 监听增量索引（P5）

随 Agent 启动；`watchdog` debounce + flock；`--no-watch-index` 关闭。

---

### `main.py` — 命令行入口

| 职责 | 说明 |
|------|------|
| CLI | 解析参数、加载配置、构建 Agent、打印回复 |
| 模式 | 说明 |
|------|------|
| 默认（无 `--once`） | 交互会话：多轮对话，输入 `exit` / `quit` / `q` 退出；自动启用 MemorySaver |
| `--once` / `-1` | 单轮：执行一条消息后退出 |
| `message` | 可选首条消息；交互模式下发送后继续循环 |
| `--thread-id` | 指定会话 ID；交互模式未指定时每次启动生成新 ID |
| `--memory` | 仅 `--once` 时可选启用记忆 |
| `-C` / `--workspace` | 工作区根目录（文件工具沙箱），默认启动时 cwd |
| `-w` / `--write` | 允许写入；**默认只读**，不注册写工具 |
| `--no-spill` | 禁用大工具结果落盘（P6） |
| `--no-watch-index` | 不随 Agent 启动文件监听索引 |
| `--quiet` / `-q` | 关闭过程展示，仅输出最终回复 |

安装后可通过 `llgraph` 命令或 `python -m llgraph` 调用（见 `pyproject.toml` 的 `project.scripts`）。

## 典型调用链（一次对话）

1. 用户执行 `llgraph` 进入交互，或 `llgraph "现在几点？"` 带首句
2. `main.py` 调用 `load_llgraph_env()` / `build_agent(with_memory=True)`
3. `agent.py` 通过 `llm.py` 创建 Gateway LLM，通过 `tools.py` 获取工具列表
4. LangGraph 执行 ReAct：模型若需实时信息则调用 `get_current_utc_time`，再生成最终回答
5. `invoke_agent()` 提取最后一条 assistant 消息并打印

## 扩展建议

| 目标 | 建议修改 |
|------|----------|
| 增加工具 | `tools.py` |
| 改系统人设 / Agent 类型 | `agent.py` 中 `_SYSTEM_PROMPT` 或替换 `create_react_agent` |
| 换模型或网关 | `~/.config/llgraph/llgraph.env` 或项目 `.env` |
| 对外提供 HTTP API | 新建模块（如 `server.py`），内部复用 `build_agent` / `invoke_agent` |
| 接入 LangSmith  tracing | 设置环境变量 `LANGSMITH_TRACING=true` 等（无需改代码即可尝试） |

## Code Index（`llgraph/code_index/`）

设计说明见 **[code-index.md](code-index.md)**（切块、RRF、重建、日志排障）。

| 模块 | 职责 |
|------|------|
| `paths.py` | `.llgraph/index/` 路径常量 |
| `index_logging.py` | 索引日志（`logs/index-*.log`、`latest.log`） |
| `rebuild.py` | `--rebuild` 清理 Lance / manifest / 可选 embed 缓存 |
| `store.py` | LanceDB 表读写、向量检索 |
| `file_scan.py` / `chunker.py` | 扫描与行窗口切块 |
| `ast_chunker.py` | 可选 tree-sitter AST 切块 |
| `embedder.py` | Gateway `/v1/embeddings` + SQLite 缓存 |
| `manifest.py` | 文件 SHA256 增量 |
| `indexer.py` | `llgraph index` 编排 |
| `search.py` / `hybrid.py` | 语义检索与 RRF |
| `index_cli.py` / `search_cli.py` | CLI 子命令 |
| `code_index_tools.py` | Agent 工具 `search_code_*` |

CLI：`llgraph index`（`--rebuild` / `--incremental` / 日志见 `logs/latest.log`）、`llgraph search`；交互：`/index`。
