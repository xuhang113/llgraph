# llgraph 操作手册

面向日常使用的命令、会话内操作与配置说明。架构与模块职责见 [模块说明.md](模块说明.md)。

**与 Cursor 的关系**：llgraph 是 Cursor Agent 工作流在终端侧的独立实现（LangGraph + Gateway），配置用 `.llgraph/` 而非 `.cursor/`。对照表见 **[cursor-agent.md](cursor-agent.md)**。

---

## 0. 产品定位（基于 Cursor 的 Agent）

| 你在 Cursor 里习惯的 | 在 llgraph 里 |
|---------------------|---------------|
| Agent 对话改代码、搜仓库 | `llgraph -w -C <工作区>` |
| Rules / Skills | `.llgraph/rules`、`.llgraph/skills`，`/rule`、`/skill` |
| @codebase / 语义搜索 | `llgraph index` + `search_code_hybrid` |
| Apply / 局部编辑 | `search_replace`（`-w`） |
| 看本轮改了啥 | `/changes`、`/diff` |
| MCP 工具 | `.llgraph/mcp.json`（需单独配置，不复用 Cursor） |
| 长对话 / 上下文 | `/compress`、`/trace stats`、工具结果自动落盘 |

**不是**：IDE 补全、图形化 diff、Cursor Cloud Agent。详见 [cursor-agent.md](cursor-agent.md)。

## 1. 安装

```bash
cd /path/to/llgraph
python3.12 -m venv .venv
source .venv/bin/activate
pip install -U pip -e '.[index,watch,mcp]'
```

网关配置（llgraph 专用，与 Claude CLI 无关）：

```bash
mkdir -p ~/.config/llgraph
cp examples/llgraph.env.example ~/.config/llgraph/llgraph.env
# 编辑 LLGRAPH_API_BASE_URL、LLGRAPH_API_KEY、LLGRAPH_MODEL
```

也可在 llgraph 项目根 `.env` 中设置同名变量（会覆盖）。

---

## 2. 首次使用工作区

```bash
cd /path/to/your-workspace
llgraph --init-config -C .
```

会在工作区生成 `.llgraph/`（rules、skills、agent.json、embedding.json 等）。**不读取** `.cursor/` 配置。

构建代码索引（可选，推荐）：

```bash
llgraph index -C .
```

---

## 3. 启动 Agent

| 命令 | 说明 |
|------|------|
| `llgraph -C <工作区>` | 交互会话（默认只读） |
| `llgraph -w -C <工作区>` | 允许改代码（`search_replace` / `write_file`） |
| `llgraph --once -C . "问题"` | 单轮后退出 |
| `llgraph --trace all` | 启动即完整过程展示 |
| `llgraph --no-watch-index` | 不自动监听文件增量索引 |
| `llgraph --no-spill` | 禁用大工具结果落盘（调试） |

交互中输入 `exit` / `quit` / `q` 退出。

## 4. 会话内命令

这些命令**不会**发给模型。

### 4.1 帮助与展示

| 命令 | 作用 |
|------|------|
| `/help` | 完整帮助 |
| `/paste` 或 `/p` | 多行粘贴（单独一行 `---` 结束） |
| `/trace` | 过程展示模式说明 |
| `/trace all\|steps\|reply\|none` | 切换展示档位 |
| `/trace stats` | token 估算 + 工具落盘统计 |

### 4.2 规则与技能

| 命令 | 作用 |
|------|------|
| `/rule` | 列出 `.llgraph/rules` |
| `/rule on\|off <id>` | 强制启用/禁用规则 |
| `/skill` | 列出技能 |
| `/skill <name>` | 启用技能 |

### 4.3 代码索引

| 命令 | 作用 |
|------|------|
| `/index` | 查看索引状态 |
| `/index full` | 全量索引 |
| `/index incremental` | 增量索引 |
| `/index rebuild` | 重建索引 |

CLI 等价：`llgraph index -C .`、`llgraph search "关键词" -C .`

### 4.4 上下文与评审

| 命令 | 作用 |
|------|------|
| `/compress` | 手动压缩对话历史 |
| `/review [主题]` | 评审本会话改动，落盘 `~/llgraph-review/` |

自动压缩：当估算 token 超过 `context.auto_compress_ratio` 阈值时触发。压缩采用 Tier1（按 token 保留最近消息且保持 tool 链完整）+ Tier2（结构化 `conversation_anchor.json` 增量合并）+ Tier3（可选 `search_code_hybrid` 补全相关代码）；配置见 `keep_recent_token_ratio`、`compress_retrieval_enabled`。

### 4.5 会话切换与清理

| 命令 | 作用 |
|------|------|
| `/sessions`、`/session` | 列出可恢复会话（标题 + thread_id） |
| `/session use <id>` | 切换到已有会话 |
| `/session new` | 新建会话 |
| `/session title <标题>` | 重命名（≤30 字） |
| `/session delete <id>` | 删除指定会话 |
| `/session delete empty` | 删除空壳（无对话仅 manifest） |
| `/session delete all` | 删除除当前外全部 |
| `/session delete all --including-current` | 全量删除并切到新会话 |

启动：`llgraph --list-sessions -C .`、`--thread-id cli-xxx`。

### 4.6 会话变更（需 `-w`）

| 命令 | 作用 |
|------|------|
| `/changes` | 本会话改过的文件 |
| `/diff <path>` | 首次编辑前快照 vs 当前文件 |
| `/changes clear` | 清空内存列表（落盘保留） |
| `/changes reset` | 清空内存并删除落盘记录 |

## 5. 配置文件（`.llgraph/`）

| 文件 | 用途 |
|------|------|
| `agent.json` | Thought、编辑、上下文压缩、评审 |
| `embedding.json` | 向量索引与 watch |
| `mcp.json` | MCP Server（**仅 llgraph 独立配置**） |
| `rules/*.mdc` | 规则 |
| `skills/<name>/SKILL.md` | 技能 |
| `commands/*.md` | 自定义 `/命令` |
| `thought/*.md` | Thought 规范 |

### 5.1 MCP（`.llgraph/mcp.json`）

与 Cursor **完全独立**，需自行填写 `servers`：

```json
{
  "defaults": { "timeout_sec": 90, "allow_write_tools": false },
  "servers": {
    "filesystem": {
      "enabled": true,
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/path/to/workspace"]
    }
  }
}
```

工具名格式：`mcp__<server>__<tool>`。

### 5.2 Shell 执行

`agent.json` → `shell` 段（默认开启）：

- 工具：`run_shell_command(command, working_directory=".")`
- 只读：允许 `pwd`、`git status`、`mvn test`；禁止 `rm`、`git commit`、shell 重定向
- `-w`：允许更多写操作，仍拦截 `rm -rf /` 等高危命令
- 日志：`.llgraph/context/terminals/commands.jsonl`

### 5.3 动态上下文 / 落盘（P6）

`agent.json` → `context` 段：

```json
{
  "tool_result_max_chars": 10000,
  "tool_result_preview_lines": 40,
  "spill_dir": ".llgraph/context/tool-results",
  "spill_enabled": true
}
```

- 内置工具（`read_file`、`grep_files`、`search_code_*`、MCP 等）返回超过阈值时，全文写入 `spill_dir`
- 模型收到的是**路径 + 行数 + 末尾预览**；需要全文时用 `read_file` / `grep_files`
- `/trace stats` 查看 `messages_tokens` 与 `spilled_bytes_on_disk`
- `.llgraph/` 默认不参与代码向量索引

工作区内（随仓库）：

```
.llgraph/context/
  tool-results/    # 大工具输出
```

用户目录（会话记忆，不占仓库）：

```
~/.llgraph/context/<工作区名>/sessions/<thread_id>/
  messages.jsonl              # 对话正文
  meta.json                   # 标题等
  manifest.json               # Skill/Rule 锚点
  conversation_anchor.json    # 结构化压缩锚点
  edits.jsonl                 # 改动账本（-w）
  snapshots/                  # /undo 快照
../<thread_id>.jsonl          # /compress 归档（与会话目录同级）
```

**会话命令**：`/sessions`；`/session use|new|title`；`/session delete <id>|empty|all`（全删加 `--including-current`）。  
CLI：`llgraph --list-sessions`、`--delete-session`、`--purge-sessions --including-current`。

旧版 `<工作区>/.llgraph/sessions/` 会在加载时自动迁到用户目录。

### 5.4 自定义命令

在 `.llgraph/commands/` 下创建 `*.md`：

```markdown
---
name: deploy-check
aliases: [dep]
description: 检查发布清单
handler: prompt
requires_write: false
---
请根据工作区 rules 检查当前变更…
```

`/commands` 或 `/help` 查看列表。

## 6. 典型工作流

### 6.1 只读排查

```bash
llgraph -C /path/to/workspace
> 拼团关团 deadline 逻辑在哪？
# Agent 会 search_workspace / search_code_hybrid / read_file
```

### 6.2 改代码

```bash
llgraph -w -C /path/to/workspace
> 帮我在 CacheRefreshListener 加一行日志
/changes
/diff services/order-api/.../CacheRefreshListener.java
/review 缓存刷新
```

### 6.3 长日志粘贴

```
> /paste
（粘贴报错栈）
---
```

### 6.4 索引维护

随 Agent 启动时默认开启 watch（`embedding.json` → `index.watch`）。改文件保存后会 debounce 增量索引。

## 7. 排障

| 现象 | 处理 |
|------|------|
| 网关 401/403 | 检查 `~/.config/llgraph/llgraph.env` 中 `LLGRAPH_API_KEY` |
| 检索无结果 | 先 `llgraph index -C .` 或 `/index full` |
| `search_replace` 失败 | 先 `read_file` 核对缩进；扩大 `old_string` 上下文 |
| token 涨太快 | `/trace stats`；确认 `spill_enabled: true`；必要时 `/compress` |
| MCP 未加载 | 检查 `.llgraph/mcp.json` 的 `servers` 是否配置 |
| 索引吃了临时文件 | 确认 `index.skip_dirs` 含 `.llgraph`（默认已内置） |

索引日志：`.llgraph/index/logs/latest.log`
